I build low-level and research-oriented software with an emphasis on deterministic behavior, inspectable execution, strong boundaries and long-term maintainability.

## Engineering defaults

| Property | Default |
|---|---|
| Architecture | Small explicit components with clear ownership |
| Correctness | Invariants, tests and failure-first design |
| Execution | Deterministic and reproducible where possible |
| State | Minimal, inspectable and deliberately introduced |
| Interfaces | Typed, narrow and difficult to misuse |
| Tooling | Automated formatting, linting, tests and CI |
| Documentation | Explain decisions, guarantees and limitations |
| Complexity | Earned by the problem, never by decoration |
